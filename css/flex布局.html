我们将flex布局分为两部分讲，轴和容器。

轴

  • 主轴（mian axis）
  • 交叉轴（cross axis）

容器

  • 父容器（container)
  • 子容器（item）

# 轴

# flex-direction

主轴决定子容器的摆放顺序

默认情况下，主轴的方向是从左向右的，交叉轴垂直于主轴，

交叉轴的方向只与主轴水平或垂直有关，而与从左到右或从右到左无关，因为当flex-direction为row或row-reverse时，align-items：start都是在上面。

当flex-direction为column或column-reverse时，align-items：start都是在左边。

那么接下来我们看flex-direction是如何决定主轴的。讲这个之前，我们需要明白

  • 交叉轴是由主轴决定的，主轴又是由flex-direction决定的。
  • flex-direction属性设置在父容器上，这样子才可以生效。
flex-direction: row | row-reverse | column | column-reverse

row: x轴从左往右

row-reverse: x轴从右往左

column： y轴从上到下

column-reverse： y轴从下到上

# 父容器

  • justify-content: 设置子元素在主轴方向上的对齐方式
  • align-items： 设置子元素在交叉轴方向上的对齐方式

# justify-content

justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly

测试代码：

<style>
    .wrapper {
        display: flex;
        flex-direction: row;
        justify-content: flex-end;
        width: 1000px;
        height: 800px;
        background-color: grey;
    }
    .item{
        width: 200px;
        height: 200px;
        background-color: rgb(189, 120, 120);
    }
</style>

<body>
    <div class="wrapper">
        <div class="flex1 item">#flex1</div>
        <div class="flex2 item">#flex2</div>
    </div>
</body>

flex-start： 子元素沿着主轴开始方向对齐。

flex-end： 子元素沿着主轴结束方向对齐。

**center：子元素在主轴方向居中 **

space-between： 子元素在主轴方向上两端对齐，项目之间间隔相等。

space-around： 子元素在主轴方向上均匀排列每个元素，每个元素周围分配相同的空间。

（个人解读就是在主轴居中排列，并且每个元素的左右margin都相同）

space-evenly: 子元素与边界的距离 和 子元素到子元素的距离 相同

这个属性兼容性不好，在只有一行的情况下可以通过space-between+伪元素实现

.wrapper {
    display: flex;
    justify-content: space-between;
    width: 1000px;
    height: 800px;
    background-color: grey;
}

.wrapper::before,.wrapper::after {
    content: '';
    display: block
}

# align-items

align-items: flex-start | flex-end | center | baseline | stretch

下面的图例flex-direction以row为例。

交叉轴的方向只与row和column有关，与是否反向无关。主轴为row，交叉轴方向就是从上到下。主轴为column，交叉轴方向就是从左到右。

flex-start: 在交叉轴的起点对齐

flex-end: 在交叉轴的终点对齐

center: 在交叉轴的上居中

baseline: 元素的第一行文字的基线对齐

stretch: 默认值。如果元素没有设定高度，就会默认占满。如果flex-direction为column，这个效果就失效了，默认为内容高度

flex1,2,3设定了200的宽高，666div没有任何设置,默认撑开到父容器的高度

# align-content

和align-items基本一致，都是定义子元素在交叉轴上的对齐方式。

与align-items不同的是，align-items把多行子元素分别对齐(子元素在该行对齐方式)。align-content把所有子元素当成一个整体来对齐。只有一行的情况下，items和content都是一样的效果。

所以align-content比items多了space-around,space-between，stretch

默认值为normal,跟stretch一个效果

如下图，父容器高度是250，item高度是200。item在每一行的高度中居中对齐. align-items:center

换成align-content:center 效果：把多行的item当成一个整体去对齐

# flex-wrap

设置子容器的换行方式

flex-wrap: wrap | nowrap | wrap-reverse

.wrapper {
    display: flex;
    flex-wrap: wrap;
    justify-content: space-between;
    width: 830px;
    background-color: grey;
}
.item {
    width: 200px;
    height: 200px;
    margin: 10px 0;
    background-color: rgb(189, 120, 120);
}

**wrap: **允许换行

**nowrap（默认值）： **不换行.

**wrap-reverse： **逆向换行

# flex-flow

flex-flow就是flex-direction和flex-wrap的简写形式。

flex-flow: row wrap    /*不写其中某个值就取其默认值*/
flex-flow: column nowrap

# 子容器

# flex-grow

子容器弹性伸展的比例，简单理解，就是把剩余的空间按比例分配给子容器。

如果子元素没有设定宽度，子元素的宽度按flex-grow比例分配。

如果设定了宽度，剩余的宽度按按flex-grow比例分配。

如下图，父容器宽900，子元素宽200，剩余一百，justify-content：space-between。

当flex1 的flex-grow为1，flex1就会把这些多出来的宽度全部吃掉。

当flex1，2，3，4的flex-grow分别为1，2，2，1时，表示多出来的宽度按这个比例进行分配

# flex-shrink

子容器弹性收缩的比例，默认为1。简单理解，就是当你子容器超出的部分，会按照对应的比例给子容器减去对应的值。

如下图父容器宽900px，子元素默认250px，flex-shrink分别为0，0，1，1。shrink为0表示不压缩，flex3，4按1：1进行压缩

当flex1，3，4宽度都为250px，flex-shrink分别为0，1，1。flex2 flex-shrink为1，宽度为1000px。

压缩计算方式：

1、计算权重：各shrink X 宽度之和、溢出空间

sum = 1x1000 + 1x250 + 1x250 =1500

溢出空间 = （250x3 + 1000）（总宽度） - 900（容器宽度） = 850

2、计算每个元素的压缩率： 元素shrink X 元素宽度 / sum

flex2： 1000 / 1500 = 2/3

flex3、flex4 ： 1 x 250 / 1500 = 1/6

3、计算压缩宽度 ： 压缩率x溢出空间

flex2 = 1000 - 2/3 * 850 = 433.33

flex3 = flex4 = 250 - 1/6 * 850 = 108.33

结果与上图相符。

# flex-basis

指定了 flex 元素在主轴方向上的伸缩之前的初始大小。auto即是元素本身大小

flex-basis: length | auto(默认值)

当元素没有设置宽高时，flex-basis：auto则是由元素的content决定，并且受max-min-width的影响、

感觉flex-basis、width、content的优先级特别复杂，看了好多文章，感觉没有一个统一且准确的答案。

但是可以肯定的是max-width/min-width的优先级是最高的

测试：

当子元素width设置200px时，

1、flex-basis为50px，content为300px的盒子时，最终该子元素宽度是200px，超出内容不会被隐藏，但会被其他flex-item覆盖

2、flex-basis为250px，content为300px，最终宽度是250px，超出内容不会被隐藏，但会被其他flex-item覆盖

3、flex-basis为50px，无content或content不足50px时，最终宽度是50px

4、flex-basis为250px，无content时，最终宽度是250px

当flex容器子项同时设置了width和flex-basis时，width比flex-basis大，则width充当了max-width的作用，basis起了min-width的作用

width比flex-basis小时，flex-basis充当了min-width和max-width的作用

当子元素不设置width时，

1、flex-basis为50px，content为300px，最终宽度是300px，超出内容不会被隐藏，但会被其他flex-item覆盖

2、flex-basis为310px，content为300px，最终宽度是310px

3、flex-basis为50px，无content或content不足50px时，最终宽度是50px

由此可得，当flex容器子项不设width时，flex-basis和content的值谁大取谁。（既basis起到min-width的作用，而没有max-width的作用）

当flex-basis为auto时，宽度由内容区自行撑开，如果设置了width，则以width为准（起到min-width和max-width的作用）

上述测试结论说的min-width、max-width的作用并不是等同于设置了真正的min和max-width属性，真正设置了min、max-width属性的优先级是比flex-grow、flex-shrink还要高的

有width为什么还需要flex-basis？

1、flex-basis的作用是初始化元素在主轴上的长度。主轴可以是row和column，当主轴为column时，那相对于的属性是height而不是width

2、width和flex-basis的作用并不太相同。比如，width和flex-basis单独作用的时候，和两者同时作用的时候…………。

# order

每个子元素的order默认为0，通过设置order值可以改变子元素的排序顺序，order越小，排在越前，可以为负数。

# flex

flex为flex-grow、flex-shrink、flex-basis的缩写形式。

flex:1    等价于     flex-grow:1,flex-shrink:1,flex-basis:0%    
有的浏览器因为版本不同可能会flex-basis会解析成0px，w3c的规范是flex：1 为 flex： 1 1 0

至于这个别人写出来的demo，是在flex-direction为column时，并且overflow-y：auto才会出现高度坍塌，并不是很理解，按理basis只是设置元素在主轴的初始大小，0和继承父元素的0%应该没有区别才对。

解释：如果 flex-basis 的值为百分数，且它flex容器的尺寸没有被显式设置（显示设置我的理解是没有设置数值），此时 flex-basis 的值会被解析为 content。

content值会根据flex子项的内容（指flex子项的子元素尺寸）来计算实际尺寸，多数情况下效果与 max-content值一致，就是说flex子项的子元素有多长其主轴初始值就有多长

但怎么解释只有overflow-y：auto才会出现这种情况？

# align-self

单独设置子容器如何沿交叉轴排列,可覆盖align-items

默认为auto，表示继承父元素的align-items

align-self: auto | flex-start | flex-end | center | baseline | stretch
