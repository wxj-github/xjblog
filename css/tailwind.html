# 什么是原子化？

我们平时写 css 是这样的：html指定类名，类里包含多个样式

<div class="aaa"></div>

.aaa {
    font-size: 16px;
    border: 1px solid #000;
    padding: 4px;
}

原子化：定义一些细粒度的 class，叫做原子 class，然后在 html 里直接引入这些原子化的 class

<div class="text-base p-1 border border-black border-solid"></div>

.text-base {
    font-size: 16px;
}
.p-1 {
    padding: 4px;
}
.border {
    border-width: 1px;
}
.border-black {
    border-color: black;
}
.border-solid {
    border-style: solid;
}

# 使用

# Create React App

create react app不支持自定义PostCss配置，并且与PostCss生态中很多重要工具不兼容，如PostCSS-import，推荐使用Vite脚手架

npx create-react-app my-project
cd my-project
npm install -D tailwindcss    
npx tailwindcss init   #生成 tailwindcss 配置文件- tailwind.config.js

配置tailwind.config.js

//tailwind.config.js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ["./src/**/*.{js,jsx,ts,tsx}"],    //++++
  theme: {
    extend: {
      
    },
  },
  plugins: [],
}

入口Css文件：

@tailwind base;
@tailwind components;
@tailwind utilities;

这三行分别是引入tailwind的基础样式、组件样式、工具样式的。

组件中使用tailwind,启动项目就可以看到样式了

export default function App() {

  return <div className="p-1 border border-black border-solid">
     hello tailwind
  </div>;
}

然后也是同样的入口css引入tailwind,启动项目即可

@tailwind base;
@tailwind components;
@tailwind utilities;

TailwindCSS并不会生成一个全量的样式包，而是根据具体使用到的语法生成对应的样式代码，这样可以确保打包产生的样式包是最小的。

# 修改类名前缀

tailwind的类名样式可能会和现有的冲突，可以在配置文件里修改前缀

//tailwind.config.js
module.exports = {
  prefix: 'tw-',
}

使用tailwind类名就得加上这个前缀了

export default function App() {

  return <div className="tw-p-1 tw-border tw-border-black tw-border-solid">
     hello tailwind
  </div>;
}

# 重写tailwind样式

export default function App() {

  return <div className="p-1 border border-black border-solid">
     hello tailwind
  </div>;
}

打开开发者工具，我们可以看到p-1是padding:0.25rem，可以在tailwind.config.js去修改它

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ["./src/**/*.{js,jsx,ts,tsx}"],
  theme: {
    extend: {
      padding:{
        '1':'20px',     //+++
        'xj':'66px',   //还可以重写原本不存在的类名,p-xj使用
      }
    },
  },
  plugins: [],
}

刷新网页就可以看到p-1的样式变成了padding:20px

而有的类名如text-base对应的样式是 font-size:1rem;line-height:1.5rem，配置文件里就要以数组的方式配置，要是只配置一个，可以看到line-height的样式就没有了

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ["./src/**/*.{js,jsx,ts,tsx}"],
  theme: {
    extend: {
      padding:{
        '1':'20px'
      },
      fontSize:{
        'base':['16px','32px']
      }
    },
  },
  plugins: [],
}

我们怎么知道p-1,text-base是在extend里的配置什么呢，就要在官网搜索p-1,text-base，就可以查到了

但这些都是全局的更改，有的时候你想临时设置一些值，可以用 [] 语法。

比如 text-[16px]，它就会生成 font-size:16px 的样式：

export default function App() {

  return <div className="text-[16px]">
     hello tailwind
  </div>;
}

比如 aspect-[4/3]，就是这样的样式：aspect-ratio: 4/3 ,纵横比4比3

# 响应式-断点

/** @type {import('tailwindcss').Config} */
module.exports = {
  theme: {
    screens: {   //这个是完全覆盖，如果下面没有写xl或2xl，那这些修饰符就无法使用
      'sm': '640px',
      // => @media (min-width: 640px) { ... }

      'md': '768px',
      // => @media (min-width: 768px) { ... }

      'lg': '1024px',
      // => @media (min-width: 1024px) { ... }

      'xl': '1280px',
      // => @media (min-width: 1280px) { ... }

      '2xl': '1536px',
      // => @media (min-width: 1536px) { ... }
    }
  }
}

单覆盖在 theme.extend 键下添加自定义 screens 值：

module.exports = {
  content: ["./src/**/*.{js,jsx,ts,tsx}"],
  theme: {
    extend: {
      'lg': '992px',
        // => @media (min-width: 992px) { ... }
    },
  },
  plugins: [],
}

//响应式，默认背景红色，宽度大于768时背景蓝色
<div className="bg-red-500 md:bg-blue-500">   
 hello tailwind
</div>

添加小断点：不能使用 extend，因为小断点会添加到断点列表的末尾，并且断点需要从小到大排序才能按预期工作 -宽度断点系统。

const defaultTheme = require('tailwindcss/defaultTheme')

module.exports = {
  theme: {
    screens: {
      'xs': '475px',
      ...defaultTheme.screens,
    },
  },
  plugins: [],
}

# 函数

div {

}

screen,允许你创建按名称引用断点的媒体查询

@media screen(sm) {
  /* ... */
}


/*编译为*/
@media (min-width: 640px) {
  /* ... */

# @layer @apply

类名太长且重复，可以使用@layer @apply去拓展

layer就是我们在入口文件引入的那三个东西，就是用来区分样式优先级的

  • base 层用于重置规则或应用于纯 HTML 元素的默认样式。
  • components 层用于你希望能够使用工具覆盖的基于类的样式。
  • utilities 层用于小型、单一用途的类，这些类应始终优先于任何其他样式。

# 跨项目复用

开发 tailwind 插件

//   /plugin/guang.js
const plugin = require('tailwindcss/plugin');

module.exports = plugin(function({ addUtilities }) {
    addUtilities({
        '.guang': {
            background: 'blue',
            color: 'yellow'
        },
        '.guangguang': {
            'font-size': '70px'
        }
    })
})
  • addUtilities()，用于注册新的静态工具样式
  • matchUtilities()，用于注册新的动态工具样式
  • addComponents()，用于注册新的静态组件样式
  • matchComponents()，用于注册新的动态组件样式
  • addBase()，用于注册新的基础样式
  • addVariant()，用于注册自定义静态变体
  • matchVariant()，用于注册自定义动态变体
  • theme()，用于查找用户主题配置中的值
  • config()，用于在用户的 Tailwind 配置中查找值
  • corePlugins()，用于检查核心插件是否启用
  • e()，用于手动转义用于类名的字符串

在 tailwind.config.js 里引入：

module.exports = {
  content: ["./src/**/*.{js,jsx,ts,tsx}"],
  theme: {
  },
  plugins: [
      require('./plugin/guang.js')
  ],
}

就可以使用了

<div className="guang guanguang">   
 hello tailwind
</div>
