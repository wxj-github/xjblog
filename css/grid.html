# Grid布局

Grid 布局即网格布局，是一种新的 CSS 布局模型，比较擅长将一个页面划分为几个主要区域，以及定义这些区域的大小、位置、层次等关系。号称是最强大的的 CSS 布局方案，是目前唯一一种 CSS 二维布局。

# 兼容性

https://caniuse.com/?search=grid

# Grid 布局和 flex 布局

flex 布局和 Grid 布局有实质的区别，那就是 flex 布局是一维布局，Grid 布局是二维布局。flex 布局一次只能处理一个维度上的元素布局，一行或者一列。Grid 布局是将容器划分成了“行”和“列”，产生了一个个的网格，我们可以将网格元素放在与这些行和列相关的位置上，从而达到我们布局的目的。

# 网格线

划分网格的线，称为"网格线"。Grid 会为我们创建编号的网格线来让我们来定位每一个网格元素。m 列有 m + 1 根垂直的网格线，n 行有 n + 1 根水平网格线。比如图中就有 4 根垂直网格线。一般而言，是从左到右，从上到下，1，2，3 进行编号排序。当然也可以从右到左，从下到上，按照 -1，-2，-3...顺序进行编号排序

# 容器属性

# display

元素上声明 display：grid 或 display：inline-grid 来创建一个网格容器。

声明 display：grid 则该容器是一个块级元素，

设置成 display: inline-grid 则容器元素为行内元素

# 固定的列宽和行高

.wrapper {
  display: grid;
  /*  声明了三列，宽度分别为 200px 100px 200px */
  grid-template-columns: 200px 100px 200px;
  /*  行列间距，row-gap,column-gap简写  */
  grid-gap: 5px; 
  /*  声明了两行，行高分别为 50px 50px  */
  grid-template-rows: 50px 50px;
}

repeat() 函数：可以简化重复的值。该函数接受两个参数，第一个参数是重复的次数，第二个参数是所要重复的值。比如上面行高都是一样的，我们可以这么去实现，实际效果是一模一样的

.wrapper-1 {
  display: grid;
  grid-template-columns: 200px 100px 200px;
  grid-gap: 5px;
  /*  2行，而且行高都为 50px  */
  grid-template-rows: repeat(2, 50px);
}

# fr 关键字：

Grid 布局还引入了一个另外的长度单位来帮助我们创建灵活的网格轨道。

fr 单位代表网格容器中可用空间的一等份。grid-template-columns: 200px 1fr 2fr 表示第一个列宽设置为 200px，

后面剩余的宽度分为两部分，宽度分别为剩余宽度的 1/3 和 2/3。

.wrapper-3 {
  display: grid;
  grid-template-columns: 200px 1fr 2fr;     /* 如果是2fr 1fr 2fr，就是一行平分五份，分三列，每列分别占2，1，2份  */
  grid-gap: 5px;
  grid-auto-rows: 50px;
}

注意：假设grid-template-columns:1fr 1fr 1fr，其中某一列的的实际内容宽度大于1fr，它会一直挤压其他列的宽度，剩下的宽度才按fr等分（其他列最多会被挤到只剩下列内容的宽度）

# minmax() 函数：

我们有时候想给网格元素一个最小和最大的尺寸，minmax() 函数产生一个长度范围，表示长度就在这个范围之中都可以应用到网格项目中。它接受两个参数，分别为最小值和最大值。

grid-template-columns: 1fr 1fr minmax(300px, 2fr) 的意思是，第三个列宽最少也是要 300px，但是最大不能大于第一第二列宽的两倍。代码以及效果如下：

.wrapper-4 {
  display: grid;
  grid-template-columns: 1fr 1fr minmax(300px, 2fr);
  grid-auto-rows: 50px;
}

当2fr的长度超过300px时，也就是wrapper超过600px时，列的长度计算才是1fr 1fr 2fr。

当wrapper小于600px时，因为2fr小于300px，所以第三列固定300px，而前两列用剩下的宽度按fr等比计算。

假设这个wrapper宽度只有400px，设置了grid-template-columns: 1fr 1fr minmax(300px, 2fr)，总共分了4fr，2fr为200px，所以第三列为最小值300px，剩下的100px由前两列设置的1fr 1fr等比平分为50px 50px。

注意：如果前两列的内容宽度超过50px，那一二列还是会被撑开，最好给item设置个overflow：hidden

# auto 关键字：

通过 auto 关键字，我们可以轻易实现三列或者两列布局。

grid-template-columns: 100px auto 100px 表示第一第三列为 100px，中间自适应。

display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 100vh;

如果容器有长度，则auto为剩余宽度，比如grid为400px，则第二列为200px；

如果容器没有长度，且第二列内容为空，那auto就是0px，有内容则是宽度是第二列内容撑开的宽度。

三行布局：

display: grid;
height: 100vh;
grid-template-rows: 100px auto 100px;

# auto-fill 关键字：

表示自动填充，让一行（或者一列）中尽可能的容纳更多的单元格。

grid-template-columns: repeat(auto-fill, 200px) 表示列宽是 200 px，但列的数量是不固定的，只要grid布局的元素可以放的下，就可以放置。比如这个wrapper元素为750px，那就是3列，如果是800px，在没有grid-gap的情况下，那就是4列。

.wrapper-2 {
  display: grid;
  grid-template-columns: repeat(auto-fill, 200px);
  grid-auto-rows: 50px;
}

# auto-fit关键字：

和auto-fill基本一致，都是表示让一行或一列尽可能容纳更多的单元格。不同的是，auto-fit剩余的位置会平分到每个单元格中，而auto-fill会空到最后面。

.wrapper {
    display: grid;
    background-color: grey;
    width: 750px;
    /* grid-template-columns: repeat(auto-fit, minmax(100px,1fr)); */
    grid-template-columns: repeat(auto-fill, minmax(100px,1fr));
    grid-gap: 5px;
    grid-auto-rows: 60px;
    /* justify-content: space-evenly; */
}

使用auto-fit效果：

需要注意的是，在一行能放下所有列时，如果grid-gap变为9px或更高，auto-fill和auto-fit的效果是一样的。

原因：

grid容器一共750px，列宽最小100px，在无grid-gap下最多容纳7列，但grid-gap为9px时。有6个间隔（n-1列个间隔），6*9=54px，所以剩余的只有696px，无法融入七列。只能放6列。继续计算。5 * 9=45px，750-45=705px，705/6=117.5px，符合效果预期。

而grid-gap为5px时，7列时为6个间隔，5*6=30px，750-30=720，还是可以容纳7列，所以最后一列就留白了。

总结：

列数(容器宽度/最小宽度) * 最小宽度+间隔数 * 间隔长度 < 容器长度。auto-fit和auto-fill才会有区别，否则效果一样

# grid-row-gap 、grid-column-gap 、 grid-gap

grid-row-gap 属性、grid-column-gap 属性分别设置行间距和列间距。

grid-gap 属性是两者的简写形式，第一个是行间距，第二个是列间距

提示：grid-gap, grid-column-gap, grid-row-gap,修改为 gap, columns-gap,row-gap

# gird-area（子项属性）、grid-template-areas

grid-template-areas 属性用于定义区域，一个区域由一个或者多个单元格组成.

属性接受一个或多个字符串作为值。

一般这个属性跟网格元素的 grid-area 一起使用， grid-area 属性指定项目放在哪一个区域

 <style>
        .sidebar {
            grid-area: sidebar;
        }

        .content {
            grid-area: content;
        }

        .header {
            grid-area: header;
        }


        .wrapper {
            margin: 50px;
            display: grid;
            grid-gap: 10px;
            grid-template-columns: 120px 120px 120px;
            grid-template-areas:
                ". header  header"
                "sidebar content content";
            background-color: #fff;
            color: #444;
        }

        .box {
            background-color: #444;
            color: #fff;
            padding: 20px;
            font-size: 150%;
        }
    </style>
<body>
    <div class="wrapper">
        <div class="box sidebar">sidebar</div>           <!-- 注意类名别写错 -->
        <div class="box header">header</div>
        <div class="box content">content</div>
    </div>
</body>

grid-template-areas:
                ". header  header"
                "sidebar content content";

上面代码表示划分出 6 个单元格，其中值得注意的是 . 符号代表空的单元格，也就是没有用到该单元格。

值得注意的是，像上面header和content都占了两个，最后header和content的宽度都为250，原因是列长度设定为120，还有一个行间距也会算进去，120+120+10=250

.sidebar {
  grid-area: sidebar;
}

.content {
  grid-area: content;
}

.header {
  grid-area: header;
}

以上代码表示将类 .sidebar .content .header所在的元素放在上面 grid-template-areas 中定义的 sidebar content header 区域中

除了通过命名配合grid-template-areas,还可以直接用网格线来声明区域

grid-area的顺序是row-start / column-start / row-end / column-end ,可以按照一个正方形，从右上角的点开始逆时针顺序记录
后面两个参数还可以使用span x

注意：

当有单元格的类设置了grid-area，但是在grid-template-areas布局中没有使用，布局会出现错乱。

当单元格的类没有设置grid-area，但在grid-template-areas有**.**时，不管有多少个句号，它都只出现在第一个句号的单元格位置

可以使用多个句号表示空白站位：比单个句号看着舒服

grid-template-areas:
    "header header  header"
    "content .....  ......";

违反一下规则， 布局也将失效：

每个区域只能定义一次：

grid-template-areas: "one one three three"      /*  three就属于定义了两片区域  */
                     "one one two two"
                     "three three four four"
                     "three three four four";

不能创建非矩形区域：如L型或T型

grid-template-areas: "one one two two"     /*  one区域就不是矩形  */
                     "one one one one"
                     "three three four four"
                     "three three four four";

# grid-auto-columns 、 grid-auto-rows

在讲 grid-auto-columns 属性和 grid-auto-rows 属性之前，先来看看隐式和显式网格的概念

隐式和显式网格：显式网格包含了你在 grid-template-columns 和 grid-template-rows 属性中定义的行和列。如果你在网格定义之外又放了一些东西，或者因为内容的数量而需要的更多网格轨道的时候，网格将会在隐式网格中创建行和列

假如有多余的网格（也就是上面提到的隐式网格），那么它的行高和列宽可以根据 grid-auto-columns 属性和 grid-auto-rows 属性设置。它们的写法和grid-template-columns 和 grid-template-rows 完全相同。如果不指定这两个属性，浏览器完全根据单元格内容的大小，决定新增网格的列宽和行高

.wrapper {
  display: grid;
  grid-template-columns: 200px 100px;
/*  只设置了两行，但实际的数量会超出两行，超出的行高会以 grid-auto-rows 算 */
  grid-template-rows: 100px 100px;
  grid-gap: 10px 20px;
  grid-auto-rows: 50px;
}

grid-template-columns 属性和 grid-template-rows 属性只是指定了两行两列，但实际有九个元素，就会产生隐式网格。

通过 grid-auto-rows 可以指定隐式网格的行高为 50px

要是设置grid-auto-columns,还得把grid-auto-flow改为column

# grid-auto-flow

grid-auto-flow 属性控制着自动布局算法怎样运作，精确指定在网格中被自动布局的元素怎样排列。默认的放置顺序是"先行后列"，即先填满第一行，再开始放入第二行，即下图英文数字的顺序 one,two,three...。这个顺序由 grid-auto-flow 属性决定，默认值是 row。

.wrapper {
  display: grid;
  grid-template-columns: 100px 200px 100px;
  grid-auto-flow: row;
  grid-gap: 5px;
  grid-auto-rows: 50px;
}

.six {
  grid-column-start: span 2;   
  background: #ECAD9E;
}

第五个项目和第六个项目之间有个空白，这个是由于第六块的长度大于了空白处的长度，被挤到了下一行导致的。在实际应用中，我们可能想让下面长度合适的填满这个空白，这个时候可以设置 grid-auto-flow: row dense，表示尽可能填满表格。

.wrapper {
  display: grid;
  grid-template-columns: 100px 200px 100px;
  grid-auto-flow: row dense;
  grid-gap: 5px;
  grid-auto-rows: 50px;
}

grid-auto-flow: column，表示先列后行。

.wrapper-1 {
  display: grid;
  grid-auto-columns: 100px;
  grid-auto-flow: column;
  grid-gap: 5px;
  grid-template-rows:  50px 50px;
}

# justify-items 、align-items 、 place-items

justify-items 属性设置单元格内容的水平位置（左中右），align-items 属性设置单元格的垂直位置（上中下）

place-items为justify-items和align-items的缩写，顺序是align-items、justify-items，缺少第二个值，默认与第一个值相等

# justify-content 、align-content 、place-content

content不同于items的是，content指全部内容区在grid容器里的位置。

justify-content 属性是整个内容区域在容器里面的水平位置（左中右），align-content 属性是整个内容区域的垂直位置（上中下）。

place-content为justify-content和align-content的缩写，顺序为align-content和justify-content，缺少第二个值默认与第一个值相等

.container {
  justify-content: start | end | center | stretch | space-around | space-between | space-evenly;
  align-content: start | end | center | stretch | space-around | space-between | space-evenly;  
}

space-around - 每个项目两侧的间隔相等。所以，项目之间的间隔比项目与容器边框的间隔大一倍

space-between - 项目与项目的间隔相等，项目与容器边框之间没有间隔

space-evenly - 项目与项目的间隔相等，项目与容器边框之间也是同样长度的间隔

stretch - 项目大小没有指定时，拉伸占据整个网格容器

蓝线的距离是相等的

# justify-self 、align-self 、 place-self （全子项属性）

justify-self 属性设置单元格内容的水平位置（左中右），跟 justify-items 属性的用法完全一致，但只作用于单个项目

align-self同理。place-self为justify-self和align-self的缩写。

justify-items作用于grid容器，justify-self作用于单元格。

.wrapper {

# 列宽固定，列数量自适应 repeat + auto-fit + justify-content

.wrapper {
    display: grid;
    grid-template-columns: repeat(auto-fill, 200px);
    justify-content: space-evenly;    /*各item距离之间距离相同，实现较好的排列效果*/
    grid-gap: 10px;
    grid-auto-rows: 60px;
}

# 列宽、数量都自适应 repeat + auto-fit + minmax

minmax弥补了上面一种方案下，将多余空白平分到每个单元格上

.wrapper {
    background-color: grey;
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
    grid-gap: 10px;
    grid-auto-rows: 60px;
}

由于每个列的宽度有可能不一样，通过设置grid-column-start: span 3，表示该单元格占3列。

这就需要加上 **grid-auto-flow: row dense;**来让后面的单元格补充前面的空白

需要注意的是，比如设置了span 3，但是屏幕最小宽度为500px，而最小宽度为200px，显然是放不下三列的，但它却不会自动换行**（我个人猜测是repeat auto-fit是尽可能的融入更多列，而span的设置让grid容器的列数最少为3了）**。就开始挤压最后的一列到仅剩内容区宽度了。

加上grid-auto-flow: row dense也会这样：

如果把4号盒子的内容加多一些，浏览器的横向滚动条就会出现了。

目前能想到的解决方法是，

1、限制浏览器的最小宽度

2、通过媒体查询动态调整盒子的span列数

3、把minmax里的最小值继续调小，以至于能融入到设置的span列数
