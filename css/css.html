# css的引入方式

<!-- 内联式（行内式），直接写在元素标签里-->
<p style="color:red">aaa</p>

<!-- 嵌入式，直接写在style标签里-->
<style>
    p{
        color:red
    }
</style>

<!-- @import -->
<style>
   @import url("./xx.css")
</style>

<!-- 外部式，通过link标签引入-->
<link rel="stylesheet" href="./xx.css" />

内联式 缺点：代码臃肿，不利于维护 优点：不会额外产生请求

嵌入式 缺点：代码复用不方便 优点：不会产生额外的请求

外部式 缺点：产生额外的请求 优点：方便维护，可以重复使用

# css元素分类

不完全举例。通过属性display可以设置

块级元素（block）: div ul ol p h1~5 talbe form

行内元素（inline）： a span label

行内块元素（inline-block）： input button img（也有说img、iframe、video等是置换元素的，即它的尺寸由渲染的内容决定，而不由标签所包含的内容决定）

块级元素特点：独占一行，可以设置宽高，如果不设置，默认为父元素的100%宽度

行内元素特点：在一行内显示，不能设置宽高，默认宽高由内容决定

行内块特点：在一行内显示，可以设置宽高

# 部分css属性

font-style	// 字体属性，默认normal、italic斜体

font-weight  // 字体粗细，默认值normal、bold加粗，还可以用数值表示。取值范围100~900. 400为normal

text-decoration  // 横线。 underline下划线，overline上划线，line-through删除线、none为无。a标签默认是下划线

text-indent  // 文本缩进。用字体大小单位

line-height   //  文本行高，用字体大小单位。设置为父元素高度可以令文本垂直居中。如果根元素body设置line-height为纯数字，比如2，表示子元素会继承这个倍数，line-height为font-size的倍数，比如p标签font-size为20px，没有设置line-height，那它就会继承得到2*20=40px

letter-spacing  //	字体间距，会影响字母和中文之间的间距

word-spacing  // 单词间距，中文之间的间距也会影响

text-align   //	文本排序方式  center居中，right右对齐，left左对齐，默认left

cursor	// 鼠标手形	默认default箭头、常用的由pointer点击手势、text竖线、wait转圈、help问号

# margin

元素外边距，设置不同个数的值时和padding的方向一样

当两个div垂直分布，分别设置了margin-bottom:30px，margin-top:100px时，会发现两个盒子之间的距离只有100px而不是130，原因是margin垂直方向上会发生外边距合并，也叫外边距塌陷

外边距塌陷只发生在块级元素之间，行内块不会发生，且是垂直方向的

计算方法：

正数&&正数 取最大的数

负数&&负数 取绝对值最大的数

正数&&负数 取相加的和

当margin为负数时：

  • margin-top为负值时，元素本身会向上移动，下面的元素会跟着向上偏移
  • margin-left为负数时，元素本身会向左移动，右边的元素会跟着向左偏移
  • margin-bottom为负数时，元素本身不会移动，但自身高度会缩减相应的值，下面的元素会向上偏移
  • margin-right为负数时，元素本身不会移动，但自身长度会缩减相应的值，右边的元素会向左偏移

通过这一特性加上flex布局的自适应，可以实现两列布局中，左侧导航栏收缩的平稳动画效果。

(左侧导航栏margin-left移到屏幕外时，右侧的内容自适应填充整个屏幕)

<body>
    <input type="button" onclick="btnClick()" value="隐藏左边"></input>
    <div class="contain">
        <div class="left">1234567890</div>
        <div class="right">1234567890</div>
    </div>
</body>

<style>
    .contain {
        display: flex;
        height: 500px;
    }

    .left {
        width: 200px;
        background-color: skyblue;
        transition: all 2s;
    }

    .right {
        background-color: pink;
        flex: 1;
    }
    
     .hidden {
        margin-left: -200px
    }
</style>

<script>
    let hiddenLeft = false
    function btnClick() {
        const leftEle = document.getElementsByClassName("left")[0]
        const rightEle = document.getElementsByClassName("right")[0]
        if (hiddenLeft) {
            leftEle.setAttribute("class", "left")
            hiddenLeft = false
        } else {
            leftEle.setAttribute("class", "left hidden")
            hiddenLeft = true
        }
    }
</script>

outline

也是外边框，常是在表单元素获取焦点时显示，如果设置了outline，那不获取焦点也会显示。

和border不同的是

  • outline不占用网页空间布局，它是一种动态样式，常在元素**获取焦点（focus）或被激活（active）**的时候呈现效果，突出元素作用。border是盒模型的一部分。
  • outline的四边都必须是一样的，且必须闭合。而border可以单独设置
  • outline的边框可以通过 outline-offset 向内向外延伸，而border不行。

部分浏览器点击原生input和button后出现的边框样式就是outline的样式

这东西每个浏览器默认不一样，最好项目初始化时的重置样式时将全局outline设为none

# float

浮动，以前用于布局的属性。值有：

  • none，默认，不浮动
  • left，左浮动
  • right，右浮动

设置了浮动，该元素就脱离了标准文档流

  • 浮动的元素会互相贴靠，如果宽度不够会换行。换行后的y轴位置参考前一个浮动元素
  • 浮动元素的宽度由内容宽度决定
  • 浮动元素只能遮挡元素，而不会遮挡文本，所以可以用来做文字环绕图片的效果

浮动元素有收缩现象，比如div本来占一行，设置了float后他的宽度只有内容占据的大小

浮动元素不参与父元素高度的计算（因为脱离了标准文档流），所以会出现父元素高度塌陷的问题

浮动清除的方法：

  1. 给父元素设定高度，缺点不宜维护

  2. 内墙法,最后一个浮动元素后面放一个空的div，并设置clear属性

    <style>
    .child{
        float:right;
        width:100px;
        height:100px;
        background:red;
    }
    .clear{
        clear:both;
    }
</style>

<div>
    <div class="child">123</div>
    <div class="child">123</div>
    <div class="clear"></div>
</div>

  3. 伪元素选择器清除。在需要清除浮动的元素放一个伪元素选择器，在里面添加内容如下

    <!-- 
	伪元素选择器：dom中不存在的元素，仅在css中用来渲染，添加一些特殊的效果
	比如p::before,选择p标签前面的假元素，实际上p元素前并没有元素


	伪类选择器：概念上的类，是抽象的，表示某种状态，比如a:hover
-->
<style>
    .child{
        float:right;
        width:100px;
        height:100px;
        background:red;
    }
    .clearfix::after{
        content:"";
        display:block;
        clear:both;
    }
</style>

<div class="clearfix">
    <div class="child">123</div>
    <div class="child">123</div>
</div>

  4. overflow:hidden 或auto

    <!--
	直接给父元素设置overflow:hidden或auto，会形成一个bfc区域
-->

<style>
    .child{
        float:right;
        width:100px;
        height:100px;
        background:red;
    }
    .clearfix{
		overflow:hidden;
    }
</style>

<div class="clearfix">
    <div class="child">123</div>
    <div class="child">123</div>
</div>

# background-image

背景图

background-image:url('image/test.jpg');   //有image只能写url
background: :url('image/test.jpg') no-repeat center top;//简写
/*
	当图片比容器小的时候，默认平铺 repeat
	不平铺 no-repeat
	横向平铺 repeat-x
	纵向平铺 repeat-y
*/
background-repeat:repeat;  


/*
	用于调整图像位置
	第一个方向为x轴（正数则图片向右移动），第二个为y轴（正数则图片向下移动）
	也可以单独设置：background-position-x、background-position-y
	值可以是像素、百分比、关键字（top、bottom、left、right、center）top right右上角
	top、bottom则表明图片贴合容器的上或下，left、right表明贴合容器的左或右
*/
background-position:50px 100px;

/*
	宽百分比的值=（容器宽度-背景图宽）*百分比。高同理
*/
width:1200px;height:1300px;
//图片宽高300px,下面两个等价
background-position:30% 60%;
background-position:270px 600px;

/*
	缩放，如果宽高比不是原来的就会变形
	还可以设置为关键字
	cover: 按背景图的最短边为标准进行拉伸，到和容器的一致，将其余部分裁切
	contain: 按背景图最长边为标准进行拉伸，图片完全显示在容器里，少的部分出现白边
*/
background-size:100px 100px;

<style>
    .father{
        width: 200px;
        height: 360px;
        border: 1px solid black;
        background-image: url('./123.png'); 
        background-size: cover;
    }

</style>
<div class="father">
    <!-- 图片532*276 -->
</div>

当容器高比宽大时，可以发现，它是将背景图的最短边（此图是高）进行拉伸，当背景图高度和容器高度一致时，然后将多余的裁切

当容器宽高调换时，容器宽比高大，它还是将背景图的最短边（此图是高）进行拉伸，当背景图高度和容器高度一致时，然后将多余的裁切

其实无关容器的宽高，cover和contain的区别是以背景图的最短边或最长边来进行拉伸适配，多余的裁切或不足的留白

# object-fit

object-fit控制的是多媒体内容和元素的适应方式，通常用在img或video元素中

默认值为fill表示填充，会拉伸导致比例失调

为none时，则不对图片进行任何处理，需要通过object-position来调整显示位置

cover以最短边来拉伸适配，保证图片能覆盖容器

contain以最长边来拉伸适配，保证能完整显示整张图片，大概率会有留白

# 线性渐变（Linear Gradient）

线性渐变是在两个或多个颜色之间沿着一条直线进行过渡。可以通过指定角度或方向来控制渐变的方向。

/* 从上到下的渐变 */
background: linear-gradient(to bottom, red, yellow);

/* 从左到右的渐变 */
background: linear-gradient(to right, red, yellow);

/* 45度角的渐变 */
background: linear-gradient(45deg, red, yellow);  /* 0度是to bottom的效果，90度是to right的效果 */

/* 多个颜色的渐变 */
background: linear-gradient(to right, red, yellow, green, blue);

/* 重复渐变 */
background: repeating-linear-gradient(to right, red, yellow 25%, green 50%)

锥形渐变是从一个中心点开始沿着一个圆形路径进行颜色过渡，类似于饼图的效果

# 变形 transform

下面小标题将列出所有transform，无论哪一种transform，都是视觉效果的变化，不会影响盒子的布局，transform不会导致浏览器回流和重绘，因此效率极高

# translate平移

translate可以让盒子在原来位置上产生位移，类似相对定位

.test{
    // translateX在水平方向上平移，translateY在垂直方向上平移，正数为右/下，负数为左/上
    transform:translateX(10px)  translateY(10px)
    // translate(10px,20px)   这种是两个方向的参数一起写
     // translate(-50%,-50%)    百分比是相对于自身的宽高
}

# scale缩放

使盒子在原来的尺寸上发生缩放

.test{
    transform: scale(1.5)   // 变为原来的1.5倍，也能写两个值，分别表示X轴方向和Y轴方向上的缩放
        //也可以单个方向进行缩放
        // scaleX(1.2) scaleY(3)
}

像下拉框的动画就可以通过transform的scaleY+改变过渡原点来实现

# rotate旋转

.test{
    transform: rotate(45deg)  //顺时针旋转45度
        
        // rotate(0.5turn)   顺时针旋转半圈,一圈360度
}

# 改变变形原点

变形原点的位置，会影响到具体的变形行为

默认情况下，变形的原点在盒子中心，你可以通过transform-origin来改变它

.test{
	transform-origin：center;	/*设置原点在盒子中心*/    
	transform-origin:left top;		/*设置原点在盒子左上角*/
    transform-origin:right bottom; /*设置原点在盒子右下角*/
    transform-origin：30px 60px；/*设置原点在盒子坐标（30，60） 位置*/
}

# 多种变形叠加

一次性设置多种变形效果

.test{
    // 旋转45度，再平移（100，100)
    transform:rotate（45deg) translate（100px,100px)
    //先平移（100，100）.再旋转45度
    transform:translate（100px,100px) rotate（45deg)
}

以上两个效果最终的位置是不一样的，因为旋转之后，x、y轴的方向也会跟着旋转，那平移的方向就改变了

# 过渡和动画

过渡：

transition:transition-property transition-duration transition-timing-function transition-delay
  • 过渡属性：针对哪个Css属性应用过渡(数值类属性)，比如 transform、width、height，若填all或不填，则针对所有css属性都应用过渡
  • 持续时间：变化所持续的时间，需要带上单位，比如3s、500ms
  • 过渡属性：本质是css属性变化的贝塞尔曲线函数，通常直接使用预设值：
    • ease-in-out：平滑开始，平滑结束 由慢到快再到慢
    • linear：线性变化
    • ease-in：仅平滑开始 由慢到快
    • ease-out：仅平滑结束 由快到慢
    • ease：缓动过渡（默认值，慢-快-慢）
  • 过渡延迟：和持续时间设置差不多
.box {
  width: 100px;
  height: 100px;
  background-color: blue;
  transition-property: width, height;
  transition-duration: 1s, 2s;     //可以这么分别单独设置width和height的持续时间
}

# BFC

B: box即盒子，页面的基本构成元素。分为inline、block和inline-block三种类型的Box

FC: Formatting Context是w3c的规范中的一种概念。它是页面中的一块渲染区域，并且有一套渲染规则，它决定了其子元素将如何定位，以及和其他元素的关系和相互作用,有BFC和IFC（Inline formatting context）

BFC的定义：

BFC（Block formatting context)直译为“块级格式化上下文”。

它是一个独立的渲染区域，它规定了内部的Block-level-Box如何布局

内部元素的渲染不会影响外界

BFC的一些重要布局规则：

  1. Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠(外边距塌陷)
  2. BFC区域不会与外面的float元素重叠。
  3. BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，反之亦然
  4. 计算BFC的高度时**，浮动元素也参与计算**

形成BFC的常见条件

  • 浮动元素、float不是none
  • 绝对定位元素、position是absolute或fixed
  • 块级元素overflow 不是visible
  • display为flex、grid、inline-block

BFC常见的应用就是清除浮动

# css的继承性

CSS 的继承性指的是当一个元素没有指定某个属性时，它会继承父元素的属性值。这使得我们可以在文档中更轻松地控制样式，减少了重复定义的需要。

CSS 中的继承性是有限的，只有某些属性可以继承。通常，文本相关的属性更容易被继承，而盒子模型相关的属性则不太容易被继承。

而继承来的属性，权重极低，可看作0

# 尺寸百分比

百分比是一个相对单位，主要是找出它的参考系是谁。

  • 普通元素（非absolute或fixed定位的元素）的参考系是父元素
  • 绝对、固定定位的元素参考系为其最近的已定位的祖先元素（relative、absolute、fixed、sticky）的padding+宽/高 (padding-left+padding-right+width)或（padding-top+padding-bottom+height）
css属性百分比相对于
width参考系的宽度
height参考系的高度（参考系高度受本身高度影响时，设置无效;比如父元素高度是不确定的，由子元素内容撑开，子元素高度设置100%那就无意义）
padding参考系的宽度
border参考系的宽度
margin参考系的宽度

如果没有找到已定位祖先元素，那么元素的高度将相对于初始包含块（initial containing block）的高度来计算。

初始包含块通常是整个视窗（viewport），可通过window.innerHeight、document.documentElement.clientHeight

# 定位position

通过position设置

  • static 静态定位。默认静态定位。标准文档流。

  • relative 相对定位。

    不脱离标准文档流。以自身原来的位置为参考点。可通过设置top、bottom、left、right进行偏移。

    作用：微调元素、做绝对定位的参考-子绝父相

  • absolute 绝对定位

    会脱离标准文档流，会出现压盖现象，可通过z-index调整，数值越大，显示在最上面。

    以非static祖先元素为参考点，如果没有则以首屏页面为参考点。

    设置绝对定位后，会发现像span之类的行内元素可以设置宽高了。

  • fixed 固定定位

    会脱离标准文档流。如果只设置固定定位，而不设置数值，他会固定在原来的位置，一旦设定数值，则以网页左上角为参考点。

  • sticky 粘性定位

    不会脱离标准文档流。

    父元素的overflow必须是visible，不能是hidden或auto。

    父元素的高度不能低于sticky元素的高度。

    如果父元素没有设置定位（relative、absolute、fixed），则相对于屏幕进行定位。否则以定位的父元素为参考点。

    设置阈值：指定top、bottom、left、right其中之一才可使粘性定位生效。top、bottom同时设置时，top优先级高；left、right同时设置时，left的优先级高。

    通常用来做头部导航栏固定

    <style>
  body {
    margin: 0;
  }
  .header {
    background-color: #333;
    color: #fff;
    padding: 10px 0;
    text-align: center;
    position: sticky;
    top: 0;
    z-index: 1000; /* 使头部位于其他内容的上层 */
  }
  .content {
    padding: 20px;
    height: 2000px; /* 使内容产生滚动 */
  }
</style>

# z-index

z-index要想有效，必须满足以下条件：

  • 元素的position属性必须设置为absolute、fixed、relative中的一种(即非静态定位)
  • 元素必须具有显示效果，即其display属性不能为none
  • 父元素的z-index必须小于子元素的z-index
  • 元素的z-index值必须为整数，负数无效

默认为auto。数值为整数，数值越大，层级越高，1也大于auto

<style>
    .father{
        width: 100px;
        height: 50px;
        border: 1px solid black;
    }
    div:nth-of-type(1){
        background-color: skyblue;
    }
    div:nth-of-type(2){
        background-color: yellowgreen;
        margin-top: -30px;
        margin-left: 20px;
    }
</style>
<div style="position:relative;z-index: 15;" class="father">
    <p>A</p>
</div>
<div style="position: relative;z-index: 10;" class="father">
    <p style="position:relative;z-index:25">B</p>
</div>

可以看到，B还是被第一个div给盖住了，因为第二个div里p的z-index比父元素的高，所以并没有生效。也可以理解为，如果最外层元素有z-index，则比较的是最外层的z-index

如果两个同级的元素z-index是相同的，那后面的元素会盖过前面的

# 盒子居中

# 水平居中

  • 设置盒子margin: 0 auto;原理：设置margin-left为auto时，margin-left就会被设置为有多大就设置多大，所以盒子会跑到最右边。同理设置margin-left为auto时，盒子会跑到最左边，所以左右margin都是auto时，auto就居中了。需在盒子display为block或flex时才生效。
  • 盒子定位为绝对定位，父元素定位为相对定位（子绝父相）。left、right都设为0，margin:0 auto;
  • 盒子定位为绝对定位，父元素定位为相对定位（子绝父相）。left: 50%; transform:translateX(-50%);
  • 父元素display:flex; justify-content:center;

# css var变量

CSS变量通常在:root伪类中定义，这样可以在整个文档中全局使用。定义变量时，需要使用两个连字符开头，并且使用 -- 前缀

使用CSS变量时，需要使用 var() 函数,var函数可以接受一个可选的第二个参数，用于在变量未定义时提供降级值，相当于默认值

:root {
    --primary-color: #3498db;
    --secondary-color: #2ecc71;
    --font-size: 16px;
}


p{
    color:var(--primary-color);
    color: var(--undefined-color, black);
}

CSS变量的一个强大特性是它们的动态性，可以通过JavaScript来改变：

function changeColor() {
    document.documentElement.style.setProperty('--main-bg-color', 'lightblue');
}

# 媒体查询

通过媒体查询，可以实现响应式设计，使得网页能够适应不同尺寸的设备和不同的浏览环境。

媒体查询通常包含一个媒体类型和一个或多个媒体特性，如果满足了媒体查询的条件，对应的样式规则将被应用。

@media media_type and (media_feature) {
    /* 样式规则 */
}
  • media_type：指定了媒体类型，如 screen（屏幕）、print（打印）、speech（语音）、all（所有媒体）等。可以省略不写，默认值是 all。
  • media_feature：指定了一个或多个媒体特性，用来匹配设备或环境的属性，如宽度、高度、颜色等。常用的媒体特性包括：
    • min-width：最小视口宽度。
    • max-width：最大视口宽度。
    • orientation：设备方向，可以是 portrait（纵向）或 landscape（横向）。
    • aspect-ratio：宽高比
@media screen and (max-width: 768px) {
    /* 当视口宽度小于等于 768px 时应用样式 */
}

@media screen and (min-width: 1024px) and (max-width: 1200px) {
    /* 当视口宽度大于等于 1024px 且小于等于 1200px 时应用样式 */
}

@media print {
    /* 当设备处于打印模式下时应用样式 */
}

@media screen and (orientation: portrait) {
    /* 当设备的方向为纵向时应用样式 */
}

/* 这样也可以 */
.contain {
    height: 500px;
    @media screen and (max-width: 1000px) {
        height:800px;
    }
}

打开f12时拖动窗口它会显示实际的窗口大小

# line-height设置1.1和110%有什么区别

应用在单个元素上效果是相同的，但line-height是可以继承的，就会影响子元素

<style>
  .two {
    font-size: 30px;
    line-height: 1.1;
    /* line-height: 110%; */
  }
  .three {
    font-size: 40px;
  }
</style>
<div class="two">
  <div class="three">666</div>
</div>

当父元素设置1.1时，子元素line-height为44px

父元素设置110%时，子元素line-height为33px

总结：

  • 带单位的：子元素数值= 父元素font-size * 继承数值
  • 不带单位的：子元素数值=子元素font-size * 继承数值
