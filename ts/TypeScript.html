传统JavaScript在写大型项目时维护难度太高，有时甚至不如重新写一遍。

JS里的变量是一个动态类型，用起来很方便，不容易报错，与此同时也带来一些安全隐患，比如有一个变量a，我们期望它是一个number，如果我们将一个字符串赋值给了a，在运算的时候就会出问题，但是它并不会报错，所以在寻找问题出处的时候就会很麻烦。

JS最大的问题是函数参数也没有类型，想传什么传什么，但是它依旧是不会报错。

TypeScript是微软设计的，就是用于解决这个问题，TS是在JS中拓展出来的，让变量从动态类型变成静态类型，提升项目的可维护性。

TS文件需要先编译成JS才能在浏览器编译运行。

# TS开发环境搭建

1、安装Node.JS node -v显示版本证明安装成功

2、npm全局安装typescript npm i -g typescript 输入tsc出一堆东西说明安装成功

3、创建一个ts文件

4、使用tsc对ts文件进行编译 tsc+文件名.ts 即可完成ts文件编译成js文件

# TS文件编译后，提示无法重新声明块范围变量

原因是在同一个文件夹下，其他文件里有同样的变量定义

方法一：删除编译后的test.js文件

方法二：在项目根目录添加配置文件tsconfig.json，内容为空的也能解决（目前最稳）

(未解决)方法三：在文件最后一行加入export {} 。在TS中，只要文件存在import或export关键字，都被视为module，这种方式虽然不会报错重复声明的问题，但是编译后的js文件无法使用，浏览器报错exports is not defined。（目前这个方法只是解决了没报错重复声明，后面会报错其他问题，所以暂时不算一种解决方法）

更详细的解决方案看这个：https://blog.csdn.net/qq_45587822/article/details/106736207

# type和interface的异同：

同：都能描述对象类型，都能进行拓展，type通过&进行拓展，interface通过extends进行拓展

不同：type还可以描述基本数据类型，interface不行。interface可以重复定义且进行合并，type不行。

可以通过interface同名合并的特性，可以对第三方库的类型进行拓展

# 泛型

在定义函数或是类时，如果遇到类型不明确就可以使用泛型

# keyof

keyof 操作符是在 TypeScript 2.1 版本引入的，该操作符可以用于获取某种类型的所有键，其返回类型是联合类型。

interface Person {
  name: string;
  age: number;
}

type K1 = keyof Person; // "name" | "age"
type K2 = keyof Person[]; // "length" | "toString" | "pop" | "push" | "concat" | "join" 
type P2 = Person[keyof Person];  // number | string
type K3 = keyof { [x: string]: Person };  // string | number
type K4 = keyof { [x: number]: Person };  // number

//至于K3为什么为string|number，我猜测是因为key值类型必须是'string', 'number', 'symbol'其中之一。
//而js中,obj[1]===obj['1'],JavaScript 在执行索引操作时，会先把数值索引先转换为字符串索引,所以我理解为在索引类型里，number也被包含在string内
type test={
    [x:string]:number; 
    [y:number]:string  //error:'number' index type 'string' is not assignable to 'string' index type 'number'.
                       //“number”索引类型“string”不可分配给“string”索引类型.按我的理解，第一行即是string|number类型的索引，其数据类型为number。第二行是索引为number类型的，值为string，冲突了所以报错
}

type test2={   //这样就不会报错了。第一行的情况就包含了第二行的情况
    [x:string]:number | string;

# extends

有时候我们定义的泛型不想过于灵活或者说想继承某些类等，可以通过 extends 关键字添加泛型约束。

interface Lengthwise {
  length: number;
}

function loggingIdentity<T extends Lengthwise>(arg: T): T {
  console.log(arg.length);
  return arg;
}

现在这个泛型函数被定义了约束，因此它不再是适用于任意类型：

loggingIdentity(3);  // Error, number doesn't have a .length property

这时我们需要传入符合约束类型的值，必须包含必须的属性：

loggingIdentity({length: 10, value: 3});

# Partial

Partial<T> 的作用就是将某个类型里的属性全部变为可选项 ?。

定义：

/**
 * node_modules/typescript/lib/lib.es5.d.ts
 * Make all properties in T optional
 */
type Partial<T> = {
  [P in keyof T]?: T[P];
};

在以上代码中，首先通过 keyof T 拿到 T 的所有属性名，然后使用 in 进行遍历，将值赋给 P，最后通过 T[P] 取得相应的属性值。中间的 ? 号，用于将所有属性变为可选。

示例：

interface Todo {
  title: string;
  description: string;
}

function updateTodo(todo: Todo, fieldsToUpdate: Partial<Todo>) {
  return { ...todo, ...fieldsToUpdate }
}

const todo1 = {
  title: "Learn TS",
  description: "Learn TypeScript",
};

const todo2 = updateTodo(todo1, {
  description: "Learn TypeScript Enum",
});

在上面的 updateTodo 方法中，我们利用 Partial<T> 工具类型，定义 fieldsToUpdate 的类型为 Partial<Todo>，即：

{
   title?: string | undefined;
   description?: string | undefined;
}

# infer
T extends U ? X : Y;

如果占位符U是一个可以被分解为几个部分的类型，如数组类型、元组、函数、字符串字面量类型等、可以通过infer来获取U类型中某个部分的类型。

infer语法的限制：

  1. infer只能在条件类型的extends子句中使用
  2. infer得到的类型只能在true语句中使用,即X

推断数组(或者元组)的类型

type InferArray<T> = T extends (infer U)[] ? U : never;  //不是数组就直接返回never

type I0 = InferArray<[number, string]>; // string | number
type I1 = InferArray<string[]>; // string
type I2 = InferArray<number[]>; // number

推断数组(或者元组)第一个元素、最后一个元素的类型

//[infer P, ... infer _]中infer P获取的是第一个元素的类型，而...infer _获取的是数组其他剩余元素的数组类型;
type InferFirst<T extends unknown[]> = T extends [infer P, ...infer _] ? P : never
type I3 = InferFirst<[3, 2, 1]>; // 3

//...infer _获取的是最后一个元素之前的所有元素类型，infer Last获取的是最后一个元素的类型。
type InferLast<T extends unknown[]> = T extends [... infer _, infer Last] ? Last : never;
type I4 = InferLast<[3, 2, 1]>; // 1

推断函数类型的参数

type InferParameters<T extends Function> = T extends (...args: infer R) => any ? R : never;

type I5 = InferParameters<((arg1: string, arg2: number) => void)>; // [string, number]

推断函数类型的返回值

type InferReturnType<T extends Function> = T extends (...args: any) => infer R ? R : never;

type I6 = InferReturnType<() => string>; // string

推断字符串字面量类型的第一个字符、最后一个字符对应的字面量类型

//First表示第一个字符，b表示剩余字符
type InferString<T extends string> = T extends `${infer First}${infer b}` ? b : [];

type cao = InferString<'sdf'>  //  'df'

# 类型守卫

**类型保护是可执行运行时检查的一种表达式，用于确保该类型在一定的范围内。**换句话说，类型保护可以保证一个字符串是一个字符串，尽管它的值也可以是一个数值。类型保护与特性检测并不是完全不同，其主要思想是尝试检测属性、方法或原型，以确定如何处理值。目前主要有四种的方式来实现类型保护：

# in关键字：

interface Admin {
  name: string;
  privileges: string[];
}

interface Employee {
  name: string;
  startDate: Date;
}

type UnknownEmployee = Admin | Employee;

function printEmployeeInformation(emp: UnknownEmployee) {
  console.log("Name: " + emp.name);
  if ("privileges" in emp) {
    console.log("Privileges: " + emp.privileges);
  }
  if ("startDate" in emp) {
    console.log("Start Date: " + emp.startDate);
  }
}

# typeof 关键字

typeof 类型保护只支持两种形式：typeof v === "typename" 和 typeof v !== typename，"typename" 必须是 "number"， "string"， "boolean" 或 "symbol"。 但是 TypeScript 并不会阻止你与其它字符串比较，语言不会把那些表达式识别为类型保护。

function padLeft(value: string, padding: string | number) {
  if (typeof padding === "number") {
      return Array(padding + 1).join(" ") + value;
  }
  if (typeof padding === "string") {
      return padding + value;
  }
  throw new Error(`Expected string or number, got '${padding}'.`);
}

# 自定义的类型保护

  • 要自定义一个类型保护，只需要简单地为这个类型保护定义一个函数即可，这个函数的返回值是一个类型谓词
  • 类型谓词的语法为 parameterName is Type 这种形式，其中parameterName 必须是当前函数签名里的一个参数名
function isNumber(x: any): x is number {
  return typeof x === "number";
}

function isString(x: any): x is string {
  return typeof x === "string";
}

function padLeft(value: string, padding: string | number) {
  if (isNumber(padding)) {
    return Array(padding + 1).join(" ") + value;
  }
  if (isString(padding)) {
    return padding + value;
  }
  throw new Error(`Expected string or number, got '${padding}'.`);
}

# declare

通过declare声明的类型或者变量或者模块，在include包含的文件范围内，都可以直接引用而不用去import或者import type相应的变量或者类型。(以此完成全局类型声明文件)

"src/**/*.ts" 表示匹配所有位于 src 目录及其子目录中的 .ts 文件。

总结起来，"src/**/*.ts" 将会包含以下文件：

  1. src/a.ts (因为它在 src 目录下)
  2. src/foo/bar.ts (因为它在 src 子目录 foo 下)
  3. src/utils/util.ts (因为它在 src 子目录 utils 下)
  4. 等等...

所以，通配符 ** 表示匹配零个或多个子目录，可以用来匹配任意深度的目录结构。而 *.ts 表示匹配所有以 .ts 结尾的文件。

//声明模块,在编辑ts文件的时候，如果你想导入一个.css/.less/.png格式的文件，如果没有经过declare的话是会提示语法错误的
declare module '*.css';
declare module '*.less';
declare module '*.png';

//声明作用域，声明完之后在其他地方的ts就可以直接API.ResponseList引用到这个接口类型
declare namespace API {
    interface ResponseList {}
}

注意(在全局声明文件中)：declare最好不要跟export同级使用，不然在其他ts引用这个.d.ts的内容的时候，就需要手动import导入了

如果是给js配置声明文件，js暴露的属性或方法，声明文件里也要export

在声明文件中，type和interface可以不加declare,效果不变

# 如何在ts项目中给引入的js文件添加类型

如果引入的是js写的库，首先尝试安装对应的类型包

npm i --save -dev @types/xxx

如果安装不到，那就在node_modules找到该包，查看index.js对外暴露了哪些方法，同级目录下创建一个index.d.ts，然后在该包的package.json中添加types:'./index.d.ts'

如果是一些单独的js文件，可直接在对应js文件同级目录生成同名.d.ts，并通过declare声明类型即可，不需要改动原代码

let count = 10;
let position = {
 	x:0,
 	y:0
}

const formatPoint = point =>{
    console.log('当前坐标：',point)
}

export { count, position,formatPoint} 

//原则上js对外暴露了多少个变量就需要声明多少个

export declare let position: {   
    x: number,
    y: number
}

export declare function add(x: number,y: number): number;

type FormatPoint = (point: number)=> void
export declare const formatPoint: FormatPoint

# axios

根据官方源码可以看到，axios平时用的get、post请求方法中有三个泛型，T，R，D, 根据上图可以推断出，

R就是axios请求返回的Promise类型对象，是返回的最外层的数据，axios已经帮我们写好默认值AxiosResponse

T就是接口返回数据的类型，因为请求返回数据类型基本和 所以我们一般只需要对这个泛型进行定义声明,

D就是config数据里的相关类型

通过以上结论可以推断出几种封装请求时的ts写法：

//登录接口
export interface loginForm {
  username: string;
  password: string;
}

//登录接口返回数据类型
export interface loginResData {
  status: number;
  data: {
    token?: string;
    message?: string;
  };
}

export interface testLoginResData{
  token?: string,
  message?: string
}

import type { loginForm, loginResData,testLoginResData } from "./type";
import { AxiosResponse } from "axios";

//方式1，在返回Promise对象里的AxiosResponse里定义泛型
export const reqLogin = (data: loginForm): Promise<AxiosResponse<testLoginResData>> => {
  return $axios.post("/api/users/login", data);
};

//方式2，在post方法里定义泛型,这种最好
export const reqLogin = (data: loginForm) => {
   return $axios.post<testLoginResData>("/api/users/login", data);
};

//方式3，与方式1类似，修改函数返回值类型，不过这种把promise返回对象的最外层属性也重写了，比较麻烦不推荐
export const reqLogin = (data: loginForm): Promise<loginResData> => {
   return $axios.post("/api/users/login", data);
};
